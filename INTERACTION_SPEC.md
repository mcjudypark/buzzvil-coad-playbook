# coad_v2 인터랙션 사양 (진단 플로우 15·16·17 + 이미지 슬라이드 오버레이)

> 이 문서는 인터랙션 로직의 "원본 기준"입니다. 슬라이드를 Figma에서 다시 만들어 재임포트해도 이 기준으로 똑같이 복원합니다. 실제 코드는 `index.html`.

## 진단 플로우 (15 → 16 → 17)

### 16P · 현 상황 진단 (점수 입력)
- **X축 = 브랜드 인지도 (올영 *밖* 시장)**: 5개 항목, 각 체크 = **4점** → hook `data-check="x0"..."x4"`
- **Y축 = 올리브영 관여도 (올영 *안*)**: 5개 항목, 각 4점 → `data-check="y0"..."y4"`
- 점수 표시 스팬: `data-score="x"` / `data-score="y"` (※ 16P 표시 점수는 16P 체크만, 회고 미포함)
- **기준:** 각 축 **3개 이상 = 높음(12점↑) / 2개 이하 = 낮음(8점↓)**, 분기선 10점
- 각 항목 문구는 "광고 없이도 / 세일 아닐 때도 / 체험단 아닌" 같은 *오거닉만* 단서 권장 (변별력)

### 15P · 세일 회고 → 진단 ±1칸(±4점) 보정
- 6문항 단일선택 키워드 (`.kw-pill`, `data-q`=질문 0~5, `data-o`=보기 index)
- **매핑 (RECAP_MAP, "질문-보기" 0-based):**
  | key | 키워드 | 보정 |
  |---|---|---|
  | `1-0` | Q2 트래픽 = 브랜드 검색·오거닉 | 인지 **+4** |
  | `3-0` | Q4 아쉬움 = 신규 유입 부족 | 인지 **−4** |
  | `0-2` / `0-3` | Q1 늘어난 것 = 랭킹 / 리뷰 수 | 관여 **+4** |
  | `1-1` / `1-3` | Q2 = 올영 추천·랭킹 / 리타겟팅·재방문 | 관여 **+4** |
  | `3-3` | Q4 = 데이터·모수 부족 | 관여 **−4** |
  | 그 외 전부 | — | 영향 없음 |
- 매핑 기준: 그 답이 *밖 인지*를 말하나 *올영 안 입지*를 말하나 + 강점(+)/약점(−)
- **최종 축점수 = 16P 점수 + 회고 보정** (0~20 clamp)
- 15P 우측 박스(#recapBox): 선택 키워드 칩 + 사분면 **성향(참고)** 표시 (두 축 부호 조합)

### 17P · 우리 브랜드 위치 (X-Y 사분면 차트)
- 차트: X=인지도(좌→우), Y=관여도(아래→위), 중심(10,10)에서 4분할. JS(`wireQuadrant`)가 그림 → 셸엔 `.qchart-host` div만 있으면 됨
- **점 위치 = (인지점수, 관여점수)** [회고 포함]
- **사분면:** 인지↑관여↑=1 점유율확장 / 인지↓관여↑=2 신뢰구축 / 인지↓관여↓=3 풀퍼널 / 인지↑관여↓=4 전환가속
- **카드 클릭 → 전략 페이지 랜딩:** `QUAD_PAGE = {1:18, 2:19, 3:21, 4:23}` (2·3·4는 2페이지 스프레드라 인트로페이지)
- **위치 강도:** 여유 = |점수−10|, 두 축 중 약한 쪽 → **10=전형형 / 6=표준형 / 2=경계형**(경계면 인접 유형 참고)
- 회고 반영 줄: "💡 세일 회고 반영 — 인지도 ↑↓ · 관여도 ↑↓" (방향만, 숫자 계산 X)

## 재임포트 워크플로
1. Figma에서 슬라이드 수정 → **"페이지 N + node-id"** 전달 (우클릭 → Copy link to selection)
2. `get_screenshot(node-id, maxDimension 3840)` → `pages/sNN.png` 교체 → **자동 반영** (조각 HTML이 그 PNG를 가리킴)
3. 위 표에서 오버레이 있는 슬라이드면 **좌표 재확인**, 없으면 PNG 교체로 끝
